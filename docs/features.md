# 功能指南

## 📊 状态监控

访问路径：`/dashboard`

实时显示 NapCat 服务的运行状态：

- **连接状态**：WebSocket 连接是否正常（绿色=已连接，红色=断开）
- **登录信息**：当前登录的 QQ 号和昵称
- **运行状态**：服务是否在线、是否正常
- **版本信息**：NapCat 版本号和协议版本

点击「刷新」按钮可手动更新数据。

## 📨 事件中心

访问路径：`/events`

实时显示收到的所有事件，按类型分类：

### 事件类型

| 类型 | 图标 | 包含 |
|---|---|---|
| 消息 | 💬 | 私聊消息、群消息 |
| 请求 | 🔔 | 好友申请、入群申请 |
| 通知 | 📢 | 成员增减、禁言、戳一戳等 |

### 筛选

点击顶部标签可筛选事件类型：全部 / 消息 / 请求 / 通知

### 对话查看

点击任意消息事件，弹出对话窗口：
- 显示与该用户的所有消息记录
- 收到的消息左对齐（灰色气泡）
- 发出的消息右对齐（蓝色气泡）

## 👥 联系人管理

访问路径：`/contacts`

### 好友列表

- 显示所有好友：QQ 号、昵称、备注
- 搜索过滤：支持按昵称、QQ 号、备注搜索
- 点击好友行 → 弹出操作菜单

### 群列表

- 显示所有群：群号、群名、成员数
- 点击群 → 右侧显示群成员列表
- 点击群行 → 弹出操作菜单

### 群成员

- 点击群后右侧显示成员列表
- 显示：QQ 号、昵称/名片、角色（群主/管理）
- 点击成员 → 弹出操作菜单

### 操作菜单

点击好友/群/成员后弹出操作菜单：

**好友操作：**
- 💬 发送消息
- 🗑️ 删除好友
- ✏️ 设置备注
- ℹ️ 获取详细信息

**群操作：**
- 💬 发送群消息
- ✏️ 设置群名
- 📢 发送群公告
- 🚪 退出群聊
- 👥 获取群信息

**成员操作：**
- 💬 发送私聊消息
- 👢 踢出群聊
- 🔇 禁言
- ✏️ 设置群名片
- 🏷️ 设置专属头衔
- ℹ️ 获取详细信息

### 好友请求

切换到「好友请求」标签查看待处理的好友申请：
- 每条显示：头像、昵称、QQ 号、验证信息、时间
- 操作按钮：✅ 同意 / ❌ 拒绝

**自动同意模式**下此列表为空（请求自动处理）。

### 数据缓存

好友列表和群列表使用本地缓存：
- 首次加载从 API 获取
- 后续访问直接读缓存（更快）
- 点击「刷新」按钮强制更新

## 💬 消息调试

访问路径：`/messages`

### 发送消息

1. 选择消息类型（私聊/群聊）
2. 输入目标 ID（QQ 号或群号）
3. 输入消息内容
4. 点击「发送」

### 历史消息

1. 选择类型（私聊/群聊）
2. 输入 ID
3. 设置条数
4. 点击「查询」

## 🔧 API 调试器

访问路径：`/debugger`

Postman 风格的 API 调试工具。

### 接口选择

按分类选择接口：
- 基础：登录信息、状态、版本
- 消息：发送、撤回、历史
- 好友：列表、信息、备注
- 群管理：信息、成员、禁言、踢人
- 状态：在线状态、Cookies
- 文件：图片、语音、文件
- 互动：戳一戳、输入状态

### 参数配置

- **必填参数**：勾选框置灰，始终携带
- **可选参数**：勾选框可点击，勾选后携带
- 参数类型自动匹配（文本/数字/下拉/复选/文本域）

### 条件参数

部分参数根据其他参数值显示/隐藏：
- `send_msg` 选择「私聊」→ 显示 QQ 号，隐藏群号
- `send_msg` 选择「群聊」→ 显示群号，隐藏 QQ 号

### 原始 JSON

勾选「显示原始 JSON」可查看发送的请求体。

### 历史记录

每次请求自动记录，点击可回放。

## 📋 日志

访问路径：`/logs`

### 日志类型

| 类型 | 说明 |
|---|---|
| 请求 | 发送到 NapCat 的 API 请求 |
| 事件 | 从 NapCat 收到的事件 |
| 系统 | 连接状态、配置变更等 |

### 筛选

点击标签筛选：全部 / 请求 / 事件 / 系统

### 实时流

新日志通过 SSE 实时推送，自动滚动到底部。

### 详情查看

点击日志行展开查看完整 JSON 数据。

### 降噪

以下日志不显示在列表中（仍写入文件）：
- 心跳事件（每 30 秒一次）
- WS 原始消息

## ⚙️ 设置

访问路径：`/settings`

### WebSocket 连接

- WS 地址（带测试连接按钮）
- WS Token（掩码显示，点击查看）
- 自动重连开关
- 重连间隔配置

### HTTP API

- API 地址
- API Token

### 语音回复

三档模式选择：
- **关闭**：文本回显
- **始终语音**：TTS 语音回复
- **自动**：AI 判断（暂不可用）

**允许用户自定义**开关：
- 开启：用户可通过 `/response-type` 命令自行设置
- 关闭：统一使用全局设置

### 好友请求

- **自动同意**：收到申请自动通过
- **手动同意**：存入待处理列表

### 平台认证

- 登录 Token（掩码显示）

### 日志

- 最大日志条数
- 是否持久化到文件

## 🤖 命令系统

用户发送 `/` 开头的消息触发命令。

### 可用命令

| 命令 | 说明 |
|---|---|
| `/help` | 显示所有可用命令 |
| `/response-type voice` | 设置回复模式为语音 |
| `/response-type text` | 设置回复模式为文本 |
| `/response-type auto` | 设置为 AI 自动（暂不可用） |

### 使用示例

```
用户发送: /help
机器人回复:
📖 可用命令列表：
  /help — 查看所有可用命令
  /response-type — 设置回复模式（语音/文本）

用户发送: /response-type voice
机器人回复: ✅ 回复模式已切换为：语音回复
```

### 优先级

回复模式的优先级：
1. 全局开关「允许用户自定义」为关 → 使用全局配置
2. 全局开关为开 + 用户已配置 → 使用用户配置
3. 全局开关为开 + 用户未配置 → 使用全局配置

## 🎙️ TTS 语音回复

### 工作原理

1. 收到私聊消息
2. 判断回复模式（全局/用户配置）
3. 如果是语音模式：文本 → MiMo TTS API → 音频 → 发送语音消息

### 配置

在 `config.json` 的 `tts` 部分配置：
- API 地址和 Key
- 音色（茉莉、冰糖等）
- 风格（温柔、活泼等）

### 支持的音色

| 音色 | 语言 | 性别 |
|---|---|---|
| 茉莉 | 中文 | 女性 |
| 冰糖 | 中文 | 女性 |
| 苏打 | 中文 | 男性 |
| 白桦 | 中文 | 男性 |
| Mia | 英文 | 女性 |
| Chloe | 英文 | 女性 |
| Milo | 英文 | 男性 |
| Dean | 英文 | 男性 |

### 风格标签

基础情绪：开心/悲伤/愤怒/平静
整体语调：温柔/高冷/活泼/严肃/慵懒
音色定位：磁性/醇厚/清亮/甜美/沙哑
